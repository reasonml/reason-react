组件申明，属性以及Self

进一步阅读我们的文档内容，请务必先大致了解 ReactJS。

ReasonReact 不使用类去声明组件。组件声明的 API 会返回一个Record(注释：Reason的一个数据类型)，你只要改写其中的render的内容就可以了。

The component template is created through ReasonReact.statelessComponent("TheComponentName"). The string being passed is for debugging purposes (the equivalent of ReactJS' displayName).

举个例子，这里是 Greeting.re 文件：

let component = ReasonReact.statelessComponent("Greeting");

在 ReactJS中，你会创建一个Component Class然后再JSX中去调用它，这样的过程实际会编译成React.createElement (myClass，{prop1: 'hello'})。 在 ReasonReact中，不同于将整个“类”（也可以说是组件模板）传入一个假想的ReasonReact.createElement函数中，我们会用一个make函数：

/* still in Greeting.re */
let component = ReasonReact.statelessComponent("Greeting");

let make = (~name, _children) => {
  ...component, /* spread the template's other defaults into here  */
  render: _self => <div> {ReasonReact.stringToElement(name)} </div>
};

make 是你在写ReasonReact JSX时会调用的函数（JSX会在稍后的内容中介绍）。现在，不使用JSX去调用一个组件的方法是：

ReasonReact.element(Greeting.make(~name="John", [||])) /* the `make` function in the module `Greeting` */
/* equivalent to <Greeting name="John" /> */

make 函数将会返回一个预定义好的Component Record。 你只要改写其中一些字段，比如熟悉的 render、 initialState、 didMount 等，所有的生命周期会在后面介绍。

Note: 不要 在make函数内使用 let component ：

let make = _children => {
  ...(ReasonReact.statelessComponent("Greeting")),
  render: self => blabla
}

因为make函数会在每次JSX调用时被调用一遍，所以你会每一次都创建一个新的组件（而造成不必要的浪费）。

Props

像上面看到的，Props只是make函数中的一些 labeled arguments。（可以参考这里） They can also be optional and/or have defaults, e.g. let make = (~name, ~age=?, ~className="box", _children) => ....

最后一个props必须是children。 如果你不需要用到它，只要将它命名为_或者_children就可以了。 用下划线开头的命名如果没有被使用的话，不会触发编译器报错。

属性名称不能是ref或者key，和在React JS里面一样，它们是保留关键词。

根据示例，你可以在另外一个文件里面用<Foo name="Reason" />来生成组件。 如果没有指定，className默认值为 "box"。 age的默认值是None。 如果你想要传递一个age属性，只要像这样：<Foo name="Reason" age=20 />

关于传递Props

在ReactJS中，有时你会在一些组件中得到一个prop，并想把这个prop传递给自己的children:

<Foo name="Reason" age={this.props.age} />

这是潜在的bug的来源，因为this.props.age可能突然变成一个null但是Foo并不能处理这种情况，反过来也是一样，这个值可能是一个null，但是现在它一直不会是null，这样Foo组件中形成了不必要的检查。

在Reason里，如果你想传递一个可选的属性ageFromProps（类型是option int, 或者说None | Some int），下面的代码是不可行的：

<Foo name="Reason" age=ageFromProps />

因为当你调用Foo的是时候，age需要一个普通的int型数据，而不是option int！你需要像下面这样解决：

switch (ageFromProps) {
| None => <Foo name="Reason" />
| Some(nonNullableAge) => <Foo name="Reason" age=nonNullableAge />
}

很繁琐。不过很幸运的是，这里有一个更好的方式：

<Foo name="Reason" age=?ageFromProps />

这表示，"我理解age是一个可选的值，当我要使用它的时候我应该传一个int型的数据， 现在我想明确的传递一个完整的option。 This isn't a JSX trick we've made up; it's just a language feature! See the section on "Explicitly Passed Optional" in the Reason docs.

self

You might have seen the render: (self) => ... part in make. JavaScript中this的概念在ReasonReact中不存在（但是在Reason里是有的，因为Reason中也有类似的对象类型）。在这里，和this等价的概念叫self。 它（self）是一个包含了 state, retainedProps, handle 和reduce字段的Record，我们用这些字段定义组件的生命周期，render函数以及当我们需要的时候获取一些别的信息。 这些概念将在稍后解释。
