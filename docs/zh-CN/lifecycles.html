生命周期

ReasonReact 有类似与 ReactJS 的生命周期事件。

didMount: self => update(state)

willReceiveProps: self => state

shouldUpdate: oldAndNewSelf => bool

willUpdate: oldAndNewSelf => unit

didUpdate: oldAndNewSelf => unit

willUnmount: self => unit

注释：

  • 我们丢掉了 component 的所有前缀。
  • willReceiveProps 需要你返回的类型是 state，而不是 update state (也不是 NoUpdate/Update/SideEffects/UpdateWithSideEffects). 我们假定你总是想要在这个生命周期更新状态。 如果不是，返回以前的 state 就可以了。
  • didUpdate, willUnmount 和 willUpdate 不允许你更新state，以防止出现无限循环。
  • 不支持 willMount，请使用 didMount 。
  • didUpdate, willUpdate 和 shouldUpdate 的输入是 oldAndNewSelf record，类型是 {oldSelf: self, newSelf: self}. 这两个字段相当于ReactJS里面的 componentDidUpdate'的 prevProps/prevState/ 结合了 props/state（通过self同时得到props和state）。 willUpdate 和 shouldUpdate 也是类似。

**请注意，您应该尽可能从生命周期事件内返回 ReasonReact.NoUpdate **。 为了准备迎接React Fiber，我们会移除在生命周期里返回一个新的state的接口。 If you need to update state, simply send an action to reducer and handle it correspondingly: self.send(DidMountUpdate).

有一些生命周期（和ReactJS相比会）表现不同，可以参考下面的内容。

获取下一个或者上一个属性：retainedProps

在ReactJS里，我们有时会在组件的生命周期中使用 prevProps(componentDidUpdate) ，nextProps(componentWillUpdate) 之类的API。 ReasonReact并不会为你自动保留这些之前的属性。 为了满足这些功能，我们提供了 retainedProps API：

type retainedProps = {message: string};

let component = ReasonReact.statelessComponentWithRetainedProps("RetainedPropsExample");

let make = (~message, _children) => {
  ...component,
  retainedProps: {message: message},
  didUpdate: ({oldSelf, newSelf}) =>
    if (oldSelf.retainedProps.message !== newSelf.retainedProps.message) {
      /* do whatever sneaky imperative things here */
      Js.log("props `message` changed!")
    },
  render: (_self) => ...
};

我们提供了 ReasonReact.statelessComponentWithRetainedProps 和ReasonReact.reducerComponentWithRetainedProps 这两个方法。 其他用法和之前不包含retained-props的一样，只是让你在make函数中多定义一个 retainedProps 字段 （可以是任何类型）。

willReceiveProps

ReactJS 中的 componentWillReceiveProps 方法接受一个 nextProps 参数， 在这里我们没有 nextProps 参数，因为它就是 make 函数的参数（labeled arguments），你可以直接拿到它们。 而要获取 现在的 props，你需要用到上面 retainedProps API：

type state = {someToggle: bool};

let component = ReasonReact.reducerComponentWithRetainedProps("MyComponent");

let make = (~name, _children) => {
  ...component,
  initialState: () => {someToggle: false},
  /* just like state, the retainedProps field can return anything! Here it retained the `name` prop's value */
  retainedProps: name,
  willReceiveProps: (self) => {
    if (self.retainedProps === name) {
      ...
      /* previous ReactJS logic would be: if (props.name === nextProps.name) */
    };
    ...
  }
};

willUpdate

ReactJS 的 componentWillUpdate 中的参数 nextProps 现在是 make 函数的参数，而现在的props (this.props) 是上面的 retainedProps，可以通过 {oldSelf} 得到：

{
  ...component,
  willUpdate: {oldSelf, newSelf} => ...
}

didUpdate

ReactJS 的 prevProps 可以通过 retainedProps 拿到，需要使用 oldSelf。

shouldUpdate

与 ReactJS 的 shouldComponentUpdate 对应。
