React Ref

The Record API is in feature-freeze. For the newest features and better support going forward, please consider migrating to the new function components.

不要和 Reason 里面的 ref 搞混了，那是 Reason 语言本身的语法，我们这里讨论React Ref。

ReasonReact ref 是一个实例变量（instance variable）。 如果它表示一个自定义个组件，那么它的类型是 ReasonReact.reactRef，如果它表示一个 React DOM 元素，那么它的类型就是 Dom.element。

type state = {
  isOpen: bool,
  mySectionRef: ref(option(ReasonReact.reactRef))
};

let setSectionRef = (theRef, {ReasonReact.state}) => {
  state.mySectionRef := Js.Nullable.toOption(theRef);
  /* wondering about Js.Nullable.toOption? See the note below */
};

let component = ReasonReact.reducerComponent("MyPanel");

let make = (~className="", _children) => {
  ...component,
  initialState: () => {
    isOpen: false,
    mySectionRef: ref(None),
  },
  reducer: /* ... */,
  render: (self) => <Section1 ref={self.handle(setSectionRef)} />
};

Attaching to a React DOM element looks the same: state.mySectionRef = {myDivRef: Js.Nullable.toOption(theRef)}.

注意 为什么ReactJS refs can be null。 Which is why theRef and myDivRef are converted from a JS nullable to an OCaml option (Some/None). 当你使用 ref 的时候，你会被要求通过 switch 去处理是 null 的情况，以避免潜在的问题。

在使用 ref 的时候，你必须要参照上一节中Instance Variable的有关内容。

ReasonReact ref 只接受回调函数作为参数，ReactJS 中字符串 ref 现在是不赞成的。

我们还暴露的一个接口，可以通过 ReasonReact.refToJsObj，类型是 ReasonReact.reactRef => Js.t {..}，来将你的 ref 作为一个 JS object 返回，让你可以自由的使用。你应该仅仅在需要获取 ReactJS Component 的时候才使用这个接口。

let handleClick = (event, self) =>
  switch (self.state.mySectionRef^) {
  | None => ()
  | Some(r) => ReasonReact.refToJsObj(r)##someMethod(1, 2, 3) /* I solemnly swear that I am up to no good */
  };
